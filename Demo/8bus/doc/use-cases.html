CyPSA Use Cases (Notebook)

The following is a discussion based upon an article on how to write effective use cases. This document is a work in progress.

Who is the customer?

In this case, the customer is a utility. A variety of practitioners within the utility will use the CyPSA tool and these include but are not limited to the following:

  • IT System Administrator
  • Power Engineer
  • Visiting Auditor

What Problem are we trying to solve?

CyPSA seeks to organize and gain insight into the diverse sets of data that a critical infrastructure provider must manage. Specifically CyPSA inventories, manages, and analyzes assets and relations among those assets. A variety of interfaces are provided.

Assets

Inventory Assets and Relations: CyPSA inventories assets (both cyber and physical). This may include the cataloguing of assets through a common interface. Data sources used to generate a catalogue of assets include PowerWorld, NPView, NMap Scans, and device configurations. Depending upon the role of the person using the tool the types of assets accessed as well as the data sources through which asset information is accessed may vary.

CyPSA allows practitioners to catalogue relations among assets and these may either be manually or programatically generated.

For example, some common relations among assets include the following:

  • Topological Network Data: Which devices and assets are connected and how? Data sources for this kind of information include NMap scans, NPView topologies (via Firewall rule analysis).
  • Security Metrics Outputs: The output of various security metrics such as overall exposure. This may include the Badger as well as the CyPSA engine.
(Inventory Use Cases)

Configure Assets: CyPSA may eventually include the ability to configure assets including relays and switches. For example, a system administrator would be able to configure and alter the state of a relay via the CyPSA interface.

Annotate Assets: CyPSA also allows practitioners to manually and programatically annotate assets. Sources of information with which to annotate assets include provenance metadata regarding the data source from which the asset was loaded, vulnerability information from vulnerability databases, configuration information, and the output oa an analysis in general.
