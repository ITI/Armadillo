Abstract

This specification describes the CPTL language, defined as a dictionary of named properties and classes using W3C's RDF technology. We note that the format and scope of specification is based directly off of the FOAF Vocabulary Specification.

The Cyber Physical Topology Language (CPTL) is a project whose main objective is to provide a human-readable, machine-actionable language for practitioners and researchers to communicate information about cyber-physical systems across different domains (e.g. electrical power, enterprise networks, smart cities). The etymology of the word system (σύστημα) means an organized whole; body and as such, graphs a natural way to formally represent different assets and relations among them within a cyber-physical system.

The CPTL Vocabulary defines terms that may be employed to construct several different views or semantic graphs, on a cyber-physical system. In order to facilitate a holistic understanding of how components of CPS interact, the CPTL Vocabulary tries to provide a more efficient means to communicate different perspectives of the same system. At the very least, the CPTL Vocabulary provides practitioners and researchers with a language to communicate inventories of assets and their connections. The intent is to provide an efficient mechanism to communicate information about CPS (at the 'right' level of abstraction) to produce datasets for scientists, and data sources for information sharing requirements of auditing and government agencies.

Currently, there are several different kinds of assets and relations supported the CPTL vocabulary. The concepts defined in the ontologies listed below are used to create different semantic graphs (documented within the appropriate ontologies) that represent a way to model a given system.

  • core: Basic primitives found in different expressions throughout CPS. These include the notion of Asset as well as paths for example.
  • file system: Terms and relations that provide specifics about a given filesystem such as Directory and filesystem hierarchy relations (e.g. sym links).
  • operating system: Terms and relations typically covered in an OS course such as Process, IPC, and user account/privilege constructs
  • network: Terms and relations described in a networking course that includes network devices such as Switches, as well as protocol-specific information such as IP address.

CPTL has been evolving since 2014 and will continue to evolve.

    Class: devfs:Directory

    Directory - In computing, a directory is a file system cataloging structure which contains references to other computer files, and possibly other directories. (http://en.wikipedia.org/wiki/Directory_(computing))

    Class: devfs:File

    File - A computer file is a resource for storing information, which is available to a computer program and is usually based on some kind of durable storage. A file is "durable" in the sense that it remains available for other programs to use after the program that created it has finished executing. Computer files can be considered as the modern counterpart of paper documents which traditionally are kept in office and library files, and this is the source of the term. (http://en.wikipedia.org/wiki/Computer_file)

    Class: devfs:FilePath

    FilePath - A path, the general form of the name of a file or directory, specifies a unique location in a file system. A path points to a file system location by following the directory tree hierarchy expressed in a string of characters in which path components, separated by a delimiting character, represent each directory. (http://en.wikipedia.org/wiki/Path_(computing))

    Class: enet:Port

    Port - A port is a software construct serving as a communications endpoint in a computer's host operating system. A port is always associated with an IP address of a host and the protocol type of the communication. It completes the destination or origination address of a communications session. A port is identified for each address and protocol by a 16-bit number, commonly known as the port number. (http://en.wikipedia.org/wiki/Port_(computer_networking))

    Class: enet:Switch

    Switch - A network switch (also called switching hub, bridging hub, officially MAC bridge[1]) is a computer networking device that connects devices together on a computer network, by using packet switching to receive, process and forward data to the destination device. Unlike less advanced network hubs, a network switch forwards data only to one or multiple devices that need to receive it, rather than broadcasting the same data out of each of its ports. (http://en.wikipedia.org/wiki/Network_switch)
